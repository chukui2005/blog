# 【学习打卡】Day 1-3 — Java后端面试准备篇

> 作者：初魁 | 学习周期：Java后端面试冲刺第1-3天 | 适合人群：Java后端求职者

---

## 一、企业简历筛选规则

### 1. 学历筛选

大多数企业的第一道门槛：
- **本科起步**：985/211 优先，普通本科有机会
- **实习/校招**：对学历要求相对宽松，社招更看重经验
- 简历通过率：985/211 简历通过率约 30-50%，普通本科约 5-15%

### 2. 技术栈筛选

HR 快速扫描简历关键词：

| 优先级 | 内容 |
|--------|------|
| 必须匹配 | Java、Spring、MySQL、Redis |
| 加分项 | 微服务、消息队列、容器（Docker/K8s）、项目经验 |
| 减分项 | 罗列框架名称但不深入、技术栈过于陈旧 |

### 3. 经历筛选

- 有相关项目经验 > 无项目经验
- 有 GitHub/技术博客 > 无任何输出
- 有大厂实习/竞赛经历 > 无

## 二、简历注意事项

### 1. 必须写的内容

- **基本信息**：姓名、学历（本科/硕士）、毕业时间、联系方式
- **技术栈**：按熟悉程度分层次写，不要堆砌
- **项目经验**：至少 2 个有技术含量的项目
- **个人博客/GitHub**：有的话一定要写

### 2. 简历常见问题

- **项目描述太空洞**：不要只写"负责系统开发"，要写具体解决了什么问题、用了什么技术、达到了什么效果
- **技术栈堆砌**：把学过的东西全写上去，面试官随便挑一个问住你就完了
- **简历过长**：1-2 页为宜，挑选最有说服力的内容
- **格式混乱**：PDF 格式，命名规范：`姓名-Java后端-实习-2026届`

### 3.STAR 法则写项目经历

```
Situation（背景）：项目是什么，为什么要做
Task（任务）：你在团队中负责什么
Action（行动）：你具体做了什么，用了哪些技术
Result（结果）：最终效果如何（量化指标更好）
```

**示例**：
> "设计并实现用户积分系统，采用 Redis 缓存热点数据，将查询耗时从 200ms 降低至 20ms，支持 QPS 5000+"

## 三、应届生如何找到合适的练手项目

### 1. 项目来源

- **开源项目**：参与真实开源社区贡献（GitHub）
- **课程项目**：B站、极客时间、慕课网的实战课
- **竞赛项目**：蓝桥杯、ACM、数学建模
- **实验室项目**：导师横向/纵向课题

### 2. 项目选择原则

1. **有技术难点**：不能是 CRUD 堆砌，要有设计挑战
2. **有业务场景**：最好是有真实用户使用的场景
3. **可扩展**：架构设计要考虑后续迭代
4. **能写到简历上**：面试官会深挖项目细节

### 3. 推荐练手方向（Java后端）

| 方向 | 技术栈 | 难度 |
|------|--------|------|
| 博客/CMS 系统 | SpringBoot + MySQL + Redis | 简单 |
| 电商秒杀 | Redis + RabbitMQ + 缓存 | 中等 |
| 实时聊天 | Netty + WebSocket | 中等 |
| 权限管理系统 | SpringSecurity + JWT | 简单 |

## 四、Java程序员的面试过程

### 1. 典型流程

```
简历筛选 → 笔试/测评 → 技术一面 → 技术二面 → HR面 → 谈薪 → Offer
```

### 2. 各环节特点

| 环节 | 内容 | 时长 |
|------|------|------|
| 技术一面 | Java 基础 + 集合 + 多线程 + 项目 | 45-60分钟 |
| 技术二面 | 框架 + 数据库 + 分布式 + 系统设计 | 45-60分钟 |
| HR面 | 稳定性 / 薪资 / 职业规划 | 30分钟 |

### 3. 技术面常见题型

- **八股文**：JVM、集合、并发、框架原理
- **手撕算法**：LeetCode 中等难度为主
- **项目深挖**：你在项目中负责什么、遇到过什么问题、怎么解决的
- **系统设计**：设计一个短链接系统、秒杀系统等
