一、类与对象的概念

1.1 什么是类，什么是对象

类（Class） 是对一类事物的抽象描述，包含属性和行为。对象（Object） 是类的具体实例。

打个比方：类就像一张建筑设计图，对象就是按照这张图建造出来的房子。一张图纸可以建造出无数栋房子，每个房子的门牌号、装修风格可能不同，但它们的基本结构都来自同一张图纸。

二、成员变量与局部变量

2.1 区别总览

对比维度 成员变量 局部变量
定义位置 类中、方法外 方法内、代码块内、构造器内、参数列表
内存位置 堆（Heap） 栈（Stack）
生命周期 随对象创建而创建，随对象回收而销毁 随方法调用入栈，方法结束出栈销毁

访问修饰符 可以用 private/public/protected 不能用访问修饰符

三、封装

3.1 什么是封装

封装是面向对象三大特性（封装、继承、多态）之一。它的核心思想是：

  • 将对象的属性和实现细节隐藏起来
  • 对外提供公共的访问方法

3.2 实现封装的步骤

  1. 使用 private 修饰成员变量
  2. 提供 public 的 getter/setter 方法
  3. 在方法中可以添加逻辑控制（数据校验）

封装的好处：

  • 隐藏实现细节，使用者不需要知道内部逻辑
  • 增加数据安全性，防止外部随意修改
  • 便于维护，修改内部实现不影响外部调用

四、构造器

4.1 构造器的定义和特点

构造器（Constructor）是创建对象时调用的特殊方法，具有以下特点：

  • 方法名与类名完全相同
  • 没有返回值类型（也不写 void）
  • 通过 new 关键字调用

4.2 默认构造器

规则：如果类中没有定义任何构造器，编译器会自动生成一个无参构造器。如果定义了一个或多个构造器，编译器不再提供默认构造器。

六、static 关键字

6.1 静态变量

静态变量（类变量）被该类的所有实例共享，存储在方法区（JDK8 后的元空间），而非堆中。

6.2 静态方法

静态方法的特点：

  • 属于类，不属于对象
  • 不能访问非静态成员（变量和方法）
  • 不能使用 this 关键字
  • 通常用作工具方法

答案：

  • ① 可以运行，final 不阻止修改数组元素
  • ② 编译错误，不能修改数组引用
  • ③ 编译错误，不能修改 String 引用

常见易错点

  1. 局部变量必须初始化：成员变量有默认值，局部变量没有，使用前必须赋值
  2. this 不能在 static 中使用：静态上下文没有当前对象
  3. static 方法不能直接访问非静态成员：需要先创建对象
  4. final 引用 ≠ 内容不可变：final 数组可以改元素，final 对象可以调 setter
  5. 默认构造器消失：一旦定义了有参构造器，要保留无参构造器就得手动写
  6. this() 必须在第一行：构造器链调用必须放在构造器首行
