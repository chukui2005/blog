一、接口（Interface）

1.1 接口的基本概念

接口是一种完全抽象的”规范”，它只规定类应该做什么，而不关心怎么做。JDK8 之前接口中只能有抽象方法和常量。

二、抽象类和接口的区别

2.1 语法层面对比

对比维度 抽象类 接口
关键字 abstract class interface
继承/实现 单继承（extends） 多实现（implements）
构造器 可以有 不能有
成员变量 任意类型 public static final 常量
方法 抽象方法 + 具体方法 抽象方法 + default/static 方法
访问修饰符 无限制 默认为 public
成员变量权限 任意 只能是 public static final

2.2 设计层面对比

抽象类描述 “是什么”（is-a） 关系：一个 Cat is-a Animal。
接口描述 “能做什么”（can-do） 关系：一个 Duck can Swim。

选择原则：

  • 有状态（字段）需要共享 → 抽象类
  • 只是定义能力规范 → 接口
  • 需要多继承能力 → 接口

常见易错点

  1. 接口的变量默认是 public static final：不能是实例变量，不能修改
  2. default 方法不是 static：需要通过对象调用，不能通过接口名调用
  3. 匿名内部类不能有构造器：因为它没有类名
  4. 内部类会持有外部类引用（非静态）：可能导致内存泄漏
  5. 枚举的构造器隐式为 private：不能显式写 public
  6. @Override 不是必需的但强烈推荐：它可以在写错时给出编译错误
  7. @Override 只能用于方法：不能用于类、字段等
