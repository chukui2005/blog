一、异常体系

Java 中的异常是一个完整的类层次结构，以 Throwable 为根：

    graph TD
    T[Throwable] --> ERR[Error 不可恢复]
    T --> EXP[Exception 可处理]
    ERR --> OOM[OutOfMemoryError]
    ERR --> SOF[StackOverflowError]
    ERR --> NCF[NoClassDefFoundError]
    EXP --> RTE[RuntimeException 非受检]
    EXP --> CE[非RuntimeException 受检异常]
    RTE --> NPE[NullPointerException]
    RTE --> AIOB[ArrayIndexOutOfBoundsException]
    RTE --> AE[ArithmeticException]
    RTE --> IAE[IllegalArgumentException]
    CE --> IOE[IOException]
    IOE --> FNFE[FileNotFoundException]
    IOE --> EOF[EOFException]
    CE --> SQL[SQLException]
    CE --> CNF[ClassNotFoundException]

1.2 Exception

Exception 是程序可以处理的异常，分为两类：

  • Checked Exception（受检异常）：编译器强制处理，要么 try-catch，要么 throws
  • RuntimeException（非受检异常）：编译器不强制处理，通常是程序 bug

常见易错点

  1. finally 中的 return 会覆盖 try/catch 的 return：尽量避免在 finally 中写 return
  2. try-with-resources 的资源变量默认是 final：不能在代码块中重新赋值
  3. 异常捕获顺序：子类异常在前，父类异常在后（否则子类异常永远不会被捕获）
  4. throws 是声明，throw 是执行：不要把 throws 错写成 throw
  5. finally 不执行的两个例外：System.exit() 和 JVM 崩溃
  6. 不要捕获 Error：OutOfMemoryError 捕获了也没意义
