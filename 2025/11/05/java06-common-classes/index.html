一、String 类

1.1 不可变性

String 是 不可变对象（Immutable）。对 String 的任何修改都会创建新对象。

String 不可变的好处：

  • 字符串常量池可以安全共享
  • 线程安全（不需要同步）
  • 用作 HashMap 的 key（哈希值不变）

1.4 String/StringBuilder/StringBuffer 对比

选择原则：

  • 字符串内容不变化 → String
  • 单线程频繁拼接 → StringBuilder
  • 多线程频繁拼接 → StringBuffer

常见易错点

  1. String 使用 == 比较的是引用：一定要用 equals() 比较内容
  2. **Integer 比较值用 equals，不用 ==**：超出 -128~127 时 == 会返回 false
  3. StringBuilder/StringBuffer 的初始容量：如果知道长度，new StringBuilder(initialCapacity) 避免扩容
  4. SimpleDateFormat 线程不安全：多线程环境下要用 ThreadLocal 或改用 DateTimeFormatter
  5. asList 返回的 List 不可变：不能 add/remove，需要包装到 ArrayList
  6. 包装类参与运算时自动拆箱可能抛 NPE
