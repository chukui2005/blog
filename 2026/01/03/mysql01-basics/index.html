一、数据库基础概念

1.1 什么是数据库

数据库（Database） 是”按照数据结构来组织、存储和管理数据的仓库”。

你平时在Excel里存数据，但Excel只能单机用、不能并发、数据量大了就卡。数据库就是专业的”数据仓库”：

  • 支持多用户同时读写（并发）
  • 数据安全有保障（权限、备份）
  • 查询速度快（索引）
  • 支持复杂查询（多表联合）

1.2 关系型数据库（RDBMS）

关系型数据库用表（Table）来存储数据，表之间通过外键建立关系。

常见的关系型数据库：

  • MySQL：开源免费，使用最广（本系列用的就是这个）
  • Oracle：商业版，大厂在用
  • PostgreSQL：功能强大，近年越来越火
  • SQL Server：微软家的

1.3 表的结构

表由行（Row）和列（Column）组成：

  • 行：一条完整的记录
  • 列：一个字段（属性），有固定的数据类型

二、SQL分类

SQL（Structured Query Language，结构化查询语言）按功能分为4类：

分类 全称 作用 关键字
DDL Data Definition Language 定义数据库结构 CREATE, DROP, ALTER
DML Data Manipulation Language 操作数据 INSERT, UPDATE, DELETE
DQL Data Query Language 查询数据 SELECT
DCL Data Control Language 权限控制 GRANT, REVOKE

学习路线：DDL（建库建表）→ DML（增删改）→ DQL（查询，最重要）→ DCL（了解即可）

数值类型：

INT 4字节 ±21亿 整数就用它

类型 说明 用法

TEXT 不限长度 长文本（文章内容）

日期类型：

类型 格式 用法
DATE YYYY-MM-DD 只存日期
DATETIME YYYY-MM-DD HH:MM:SS 日期+时间
TIMESTAMP 同上，但范围小 记录修改时间

口诀：用INT存整数，DECIMAL存钱，VARCHAR存字符串，DATETIME存时间

四、约束（Constraint）

约束是限制表中数据规则的”守门员”。

约束 关键字 作用

4.1 主键（PRIMARY KEY）

每张表必须有且只有一个主键。

AUTO_INCREMENT（自增）：每次插入自动+1，不用手动指定。

建立表与表之间的关联。

外键的作用：保证数据完整性。比如不能给一个不存在的部门添加员工。

5.3 DELETE（删除）

DELETE vs TRUNCATE 的区别：

  • DELETE：逐行删除，可以加WHERE，不会重置AUTO_INCREMENT
  • TRUNCATE：直接删表重建，速度更快，重置AUTO_INCREMENT

WHERE vs HAVING：

  • WHERE：分组前过滤，不能使用聚合函数
  • HAVING：分组后过滤，可以使用聚合函数
