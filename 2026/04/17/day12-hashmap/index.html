【学习打卡】Day 12 — HashMap 面试必问 9 道题

作者：初魁 | 学习周期：Java后端面试冲刺第12天 | 适合人群：Java后端求职者

1. HashMap 的 put() 流程

  1. 计算 key 的 hashCode()，经过哈希扰动得到哈希值
  2. 用 (n-1) & hash 计算下标（桶位置）
  3. 桶头节点为空 → 直接插入新节点
  4. 头节点不为空 → 遍历链表/红黑树，用 equals() 比较 key
    • key 已存在 → 覆盖旧值
    • key 不存在 → 插入新节点（JDK 1.8 尾插法）
  5. 插入后判断链表长度是否达到树化阈值（≥8），超过则转红黑树
  6. size++，判断是否超过 capacity × loadFactor（默认 0.75），超过则扩容 2 倍

2. HashMap 容量为什么总是 2 的幂次？

为了高效计算下标位置。

(n-1) & hash = 用位运算代替取模运算。

当 n=16（2⁴）时，n-1 = 15 = 1111（二进制），任何 hash 与 1111 按位与，结果都在 0~15 之间，分布均匀。

如果是奇数或非 2 幂次，n-1 二进制会有 0，导致某些 hash 永远映射不到某些桶，碰撞增多，效率下降。

简单说：2ⁿ-1 的二进制全是 1，能让 hash 更均匀地分布到各个桶里，用 & 运算代替 % 取模，效率更高。

3. JDK 1.7 和 1.8 的 HashMap 实现有什么区别？

  1. 数据结构：1.7 是数组+链表；1.8 是数组+链表+红黑树（链表长度 > 8 时树化）

  2. 插入方式：1.7 是头插法（可能逆序、可能死循环）；1.8 是尾插法（顺序插入）

  3. 扩容后重新计算：1.7 重新计算每个元素的桶位置；1.8 利用 (n-1) & hash 判断，无需重新计算

  4. 并发问题：1.7 并发扩容可能产生死循环（环形链表）；1.8 仍有线程安全问题但不致死循环

  5. 扰动函数：1.7 多次 hash 较复杂；1.8 只用一次 (h = key.hashCode()) ^ (h >>> 16)

4. HashMap 是线程安全的吗？

不是。

多线程问题：

  • 并发 put 可能互相覆盖，数据丢失
  • 并发扩容时，JDK 1.7 会形成环形链表，get() 死循环
  • size++ 不是原子操作，size 值不准确

线程安全替代方案：

方案 原理 性能
Hashtable synchronized 方法级 差
Collections.synchronizedMap() 包装后全局锁 差
ConcurrentHashMap 分段锁 → CAS+synchronized 好 ✅
ConcurrentSkipListMap 无锁有序 好 ✅

直接用 ConcurrentHashMap，别用 HashTable。

5. ConcurrentHashMap 在 JDK 1.8 里怎么保证线程安全？

核心：CAS + synchronized，不锁整个表。

  1. 数组初始化：用 CAS 保证只初始化一次
  2. 空桶插入：用 CAS 保证只有一个线程插入成功
  3. 链表/红黑树操作：synchronized 锁住当前桶节点，允许并发度 = 桶数量
  4. 扩容：多线程协作扩容

比 Hashtable 好在哪里：

  • Hashtable 全局一把锁，所有读写互斥
  • ConcurrentHashMap 锁的是单个桶，多线程可以同时操作不同桶，并发度更高

6. JDK 1.7 多线程扩容死循环问题

JDK 1.7 HashMap 在并发扩容时，重新映射桶元素使用的是头插法。

两个线程同时扩容时，链表中的节点顺序可能被反转，形成环形链表：

JDK 1.8 改用尾插法，解决了死循环问题，但依然不是线程安全的，仍需使用 ConcurrentHashMap。

7. 红黑树特点

红黑树是一种近似平衡的二叉搜索树（不是严格平衡）。

性质：

  • 每个节点非红即黑
  • 根节点是黑色
  • 叶子节点（Nil）是黑色
  • 红节点的子节点必须是黑色
  • 从任一节点到其每个叶子的路径上，黑节点数相同

为什么需要红黑树：普通的 BST 在数据有序时会退化成链表（O(n)），红黑树通过近似平衡保证增删查操作的时间复杂度稳定在 **O(log n)**。

8. 哈希扰动函数

hash = (h = key.hashCode()) ^ (h >>> 16)

将 key 的 hashCode 高 16 位和低 16 位进行异或，增加随机性，让哈希值更分散，减少碰撞。

9. HashMap 的重要参数

参数 默认值 说明
initialCapacity 16 初始容量
loadFactor 0.75 负载因子
TREEIFY_THRESHOLD 8 链表转红黑树的阈值
UNTREEIFY_THRESHOLD 6 红黑树转链表的阈值
MIN_TREEIFY_CAPACITY 64 树化的最小容量
